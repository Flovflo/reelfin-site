Your Jellyfin tuned for Apple

ReelFin is a native Jellyfin client for iPhone, iPad, and Apple TV. It prefers Apple-safe playback paths first, then falls back to remux or Apple-optimized transcode profiles when a source needs extra compatibility work.

Built for iOS and tvOS right now. Other Apple platforms can come later, but the current product is focused on making iPhone, iPad, and Apple TV feel fast, native, and predictable first.

Playback Pipeline

Direct play first, optimized when needed

ReelFin is built around Apple playback frameworks, so the cleanest startup path is always the one Apple can decode directly. The app tries the least destructive route first and only asks the server for heavier compatibility work when it actually has to.

Direct play when Apple can decode it

Apple-safe MP4, MOV, M4V, and compatible codec combinations stay on the fastest native path.

MKV is not automatically a full transcode

Compatible MKV sources can still go through remux or direct stream instead of being re-encoded from scratch.

Apple-optimized transcode when required

When a file is less Apple-friendly, ReelFin can request safer Jellyfin outputs like clean HEVC or other compatibility profiles.

No hidden codec stack inside the app

ReelFin itself stays native. When compatibility work is needed, the server route handles it instead of a bundled third-party player.

Privacy

Clear about storage, tokens, and diagnostics.

  • Preferences and lightweight metadata stay on-device.

Playback, Plainly

Apple-safe files start fastest. Less compatible media takes more work.

  • Direct play is the quickest path when the source already matches Apple playback support.
  • MKV can still use remux or direct stream, but it is not the same as a clean Apple-native file.
  • When compatibility drops, ReelFin asks Jellyfin for safer Apple-oriented outputs, which can take longer to start.
